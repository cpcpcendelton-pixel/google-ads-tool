google-ads-tool Independent developer · Internal use
Google Ads API integration

Internal automation for my own Google Ads accounts.

A command-line tool I develop and operate to report on and manage the advertising campaigns in my own Google Ads accounts, all held under a single manager account (MCC). It replaces repetitive manual work in the Google Ads interface with scheduled, rule-based automation.

Business model

I am an independent developer who runs advertising for my own accounts. I am not an agency and I do not manage advertising for external clients. The tool described here is built solely for my own use: it reads the performance of my campaigns and adjusts them according to rules I define.

There is no product, no resale, and no redistribution. Google Ads API data never leaves my own environment and is never shared with or sold to any third party. The tool exists to save my own time and to make my campaign management consistent and measurable.

What the tool does

Reporting

Pulls daily performance — spend, conversions, and cost per acquisition — for each campaign and account under my manager account.

Campaign management

Adjusts daily budgets, target CPA, and the on/off state of campaigns and ad groups based on internal performance thresholds.

Campaign creation

Creates campaigns with their budgets, ad groups, and ads from a predefined configuration, so launches are consistent and repeatable.

Scheduling

Runs on a fixed schedule. Every change is logged, and sensitive actions run with a safety check before anything is applied.

How a typical run works

  • Authenticate against my manager account using OAuth credentials stored locally.
  • Read the enabled accounts under the manager account and pull recent performance for each campaign.
  • Evaluate each campaign against my own thresholds — for example, whether cost per install is within target.
  • Act where a threshold is exceeded: adjust the budget or target CPA, pause an underperforming ad group, or flag the campaign. Every action is recorded.
  • Log the outcome of the run so I can review what changed and why.

Google Ads API use

Account structure One manager account (MCC) with my own client accounts linked beneath it. A single developer token is used for all of them.
Reading data GoogleAdsService GAQL queries for campaign, ad group, and asset performance (cost, impressions, clicks, conversions, cost per acquisition).
Budgets CampaignBudgetService Create budgets and adjust daily budget amounts.
Campaigns CampaignService Create campaigns, set and adjust target CPA, and change campaign status.
Ad groups & ads Create ad groups and ads when launching, and change ad group status during optimization.

Data handling

All API data is used only to manage my own accounts. It is processed in memory and, at most, written to a local file for my own review. Nothing is resold, redistributed, or shared with any third party, and no external users have access to the tool.
